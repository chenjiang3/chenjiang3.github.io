冒泡排序

发表于 2019-04-25 |

简介

冒泡排序是一种简单的排序算法。它需要重复的遍历要排序的数列，一次比较两个元素，如何这两个元素顺序错误就把他们交换过来。重复这个过程知道不需要再交换，也就是该数列已经排序完成。

冒泡排序对n个元素进行排序需要O(n^2)次比较次数，并且可以原地排序。

具体算法过程如下：

1、比较相邻的元素，如果第一个比第二个大，就交换他们两个。

2、对每一个对相邻的元素做同样的工作，从开始的第一对知道结尾的最后一对，这一趟比较做完之后，最后的元素是最大的数。

3、对最后一个元素之前的n-1个数据再一次重复上面的步骤，结束之后，能求出第二大的那个元素。

4、重复上面的步骤，直到没有数据需要比较。

举个例子

假如要对【5, 1, 4, 2, 8】进行排序。算法步骤如下：

第一次循环，【5, 1, 4, 2, 8】两两比较获得最大值

1、【5, 1, 4, 2, 8】中5比1大，所以5和1交换，数列变成【1，5，4，2，8】。

2、5和4比较，5比4大，交换后数列变成【1，4，5，2，8】。

3、5和2比较，5比2大，交换后数列变成【1，4，2，5，8】。

4、5和8比较，5比8小，不用交换，最后数列为【1，4，2，5，8】。

第一次循环把最大的8找出来了，放在最后，接下来进行第二次循环，需要比较的数据是除8之外的数据，也就是【1，4，2，5】

第二次循环

1、【1，4，2，5】，1和4比较，1比4小，不用交换，还是【1，4，2，5】。

2、【1，4，2，5】，4和2比较，4比2大，交换，数列变成【1，2，4，5】。

3、【1，2，4，5】，4和5比较，4比5小，不用交换，数列还是【1，2，4，5】。

经过第二次循环，最大的5找出来了。那么第三次循环的数据便是【1，2，4】

第三次循环

1、【1，2，4】，1和2比较，1比2小，不交换，数列还是【1，2，4】。

2、【1，2，4】，2和4比较，2比4小，不交换，数列还是【1，2，4】

经过第三次循环后，4找出来了，那么第四次循环的数列是【1，2】。

第四次循环

1、【1，2】，1和2比较，1比2小，不交换，数列还是【1，2】。

所以，经过4次循环，数据已经是有序的了，顺序为【1，2，4，5，8】。
